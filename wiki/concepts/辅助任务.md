## 概念定义

**辅助任务** 是指在 [[Hermes Agent]] 的主副模型架构中，由副模型（或称辅助模型）负责处理的一类相对简单、不涉及复杂逻辑或核心决策的任务。这类任务的设计初衷是为了有效管理和优化 [[Token 消耗]]，通过将低复杂度的工作分配给成本更低、效率更高的模型来降低整体运营成本。辅助任务通常包括但不限于批准、压缩、重刷记忆、MCP 调用、Session 搜索、视觉及网页处理等。通过将这些任务从主模型中剥离，主模型可以更专注于处理复杂逻辑和核心决策，从而提升整体系统的响应速度和效率，并实现 [[性能与成本平衡]]。这种任务分工策略是 [[Hermes Agent]] 在应对用户反馈的 Token 消耗过高问题时提出的重要解决方案之一，旨在实现 [[Token 优化配置]]。

## 技术细节

在 [[Hermes Agent]] 的主副模型架构中，任务被明确地划分为由主模型处理的复杂任务和由副模型处理的辅助任务。辅助任务的配置允许用户指定一个或多个低成本模型（例如 [[Gemini 1.5 Flash]]）来承担这些工作。具体而言，辅助任务包括：

*   **批准 (Approval):** 对某些操作或输出进行初步的确认或审查。
*   **压缩 (Compression):** 对数据或信息进行压缩处理，以减少存储或传输的开销。
*   **重刷记忆 (Memory Refresh):** 更新或重置 Agent 的记忆模块，以适应新的上下文或避免信息过载。
*   **MCP 调用 (MCP Call):** 执行特定的消息通信协议调用，可能涉及与其他服务或模块的交互。
*   **Session 搜索 (Session Search):** 在当前会话或历史记录中进行搜索，以检索相关信息。
*   **视觉任务 (Visual Tasks):** 处理图像识别、分析等与视觉信息相关的任务。
*   **网页处理 (Web Processing):** 抓取、解析或处理网页内容。

通过将这些任务分配给副模型，可以显著降低主模型处理的 Token 量，从而实现更低的 [[Token 消耗]] 和更优的 [[运营成本]]。配置辅助任务通常需要在 Agent 的设置中指定相应的模型接口和参数。

## 应用场景

辅助任务的应用场景非常广泛，尤其是在需要处理大量信息、频繁交互或涉及多模态输入的场景下。例如：

*   **内容审核与过滤：** 在处理用户生成内容时，辅助模型可以先进行初步的审核和过滤，将可疑或不符合要求的内容标记出来，再由主模型进行更深入的分析。
*   **信息检索与摘要：** 当用户需要从大量文档或网页中查找信息时，辅助模型可以负责初步的搜索和信息提取，然后将关键信息摘要后呈现给主模型，由主模型进行整合和输出。
*   **多模态交互：** 在处理包含图像、视频等多种模态的输入时，辅助模型可以负责对视觉信息进行初步的解析和特征提取，为主模型提供结构化的视觉数据。
*   **自动化流程：** 在自动化工作流中，辅助任务可以处理一些标准化的、重复性的操作，例如数据格式转换、文件上传下载等，从而解放主模型去处理更具创造性和决策性的任务。
*   **成本敏感型应用：** 对于预算有限的项目或个人用户，利用低成本的副模型处理辅助任务是实现 [[免费模型利用]] 和 [[零成本上手]] 的关键。

通过合理配置辅助任务，可以显著提升 [[Hermes Agent]] 的整体效率和经济性，使其在各种应用场景下都能提供更优的 [[交互体验]] 和 [[性能与成本平衡]]。
