## 概念定义

核心智能体（Core Agent）在动态工作流（Dynamic Workflows）的架构中扮演着至关重要的角色，尤其是在复杂的多智能体协作模式下。它通常是工作流的最后一个阶段——“最终合成”阶段的执行者。与负责特定任务的子智能体（Sub-agent）不同，核心智能体的主要职责是接收来自多个并行运行的子智能体或经过“对抗验证”（Adversarial Validation）的分析结果，并将这些分散的信息进行整合、提炼和总结，最终生成一份连贯、准确且具有权威性的报告或输出。核心智能体的设计目标是能够理解和处理来自不同来源、可能存在细微差异甚至冲突的信息，并从中提取出最关键的洞察。它需要具备强大的逻辑推理能力、信息综合能力以及对最终输出格式和要求的深刻理解。在某些场景下，核心智能体甚至可以被设计为能够根据接收到的反馈动态调整其输出策略，以最大化最终报告的价值和实用性。

## 技术细节

核心智能体通常需要具备以下技术能力：

*   **信息聚合与去重：** 能够有效地收集来自不同智能体或验证环节的输出，并识别和处理重复或冗余的信息。
*   **逻辑推理与综合：** 能够理解不同信息片段之间的逻辑关系，进行推理，并将分散的信息整合成一个有机的整体。
*   **冲突解决：** 当接收到的信息存在冲突时，核心智能体需要具备一定的能力来识别冲突点，并根据预设的规则或优先级进行判断和解决，或者将冲突点明确地标记出来供用户决策。
*   **格式化输出：** 能够按照预定的格式（如报告、JSON、Markdown等）生成最终输出，确保其清晰、易读且符合预期。
*   **上下文管理：** 需要维护整个工作流的上下文信息，以便在合成最终报告时能够准确地引用和关联相关细节。
*   **与外部工具集成（可选）：** 在某些高级场景下，核心智能体可能还需要调用外部工具来辅助其完成最终的报告生成或格式化工作。

在“多智能体协作模式”中，核心智能体是连接“并行分析”和“对抗验证”阶段与最终产出的关键节点。它接收来自这些阶段的各种分析结果和验证反馈，然后进行“最终合成”，生成用户期望的输出。例如，在处理大量 GitHub Issue 时，子智能体可能负责分析每个 Issue 的具体内容和影响，而核心智能体则负责汇总所有分析结果，识别出最高优先级的 Bug，并生成一份包含这些 Bug 及其优先级排序的报告。

## 应用场景

*   **大规模 GitHub Issue 分析与验证：** 在处理海量 GitHub Issue 时，核心智能体可以汇总所有子智能体对 Issue 的分析结果，识别出 T10 级别的核心 Bug，并根据“崩溃数据”和“数据丢失”等优先级信号，生成一份优先级排序的 Bug 报告。
*   **代码审查报告生成：** 在 [[代码审查]] 工作流中，核心智能体可以整合来自多个模型（如 [[Opus]]、[[Sonnet]]、[[Haiku]] 或 [[Codex CLI]]）的代码审查意见，识别出潜在的 Bug 和安全漏洞，并生成一份综合性的代码审查报告。
*   **系统级架构分析报告：** 在对复杂系统进行分析时，核心智能体可以汇总来自不同模块或子系统的分析结果，生成一份全面的系统级架构分析报告，突出关键的风险点和改进建议。
*   **数据分析与洞察提取：** 在处理大量数据时，核心智能体能够整合不同分析智能体提取的洞察，形成一个更宏观、更深入的数据分析报告。
