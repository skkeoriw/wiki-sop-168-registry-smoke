## 概念定义

用户反馈是指用户在使用产品或服务过程中，通过各种渠道（如意见征集、评价、使用报告、直接沟通等）表达的关于产品功能、性能、易用性、稳定性、成本效益等方面的意见、建议、问题或满意度。在 [[Hermes Agent]] 的语境下，用户反馈尤为关键，它直接驱动了对 Agent 架构的优化，特别是针对 [[Token 消耗]] 过高的问题。通过分析用户在使用过程中遇到的具体场景和痛点，开发者能够识别出哪些任务是低复杂度且可以由更经济的模型处理的，从而设计出 [[主副模型架构]]，将核心决策和复杂逻辑交由主模型，而将批准、压缩、记忆重刷、MCP 调用、Session 搜索、视觉及网页处理等辅助任务分配给成本更低、效率更高的副模型。这种基于用户反馈的精细化配置，旨在实现 [[性能与成本平衡]]，提升整体的 [[交互体验]] 和 [[运营成本]] 控制。

## 技术细节

用户反馈的收集和分析是持续改进 [[Hermes Agent]] 的核心环节。当用户报告 [[Token 消耗]] 过高时，这直接指向了模型使用效率的问题。为了解决这一痛点，Hermes Agent 引入了 [[主副模型架构]]。在这种架构下，系统会根据任务的复杂度和性质，智能地选择使用主模型还是副模型。例如，处理复杂的逻辑推理和核心决策的任务会分配给主模型，而一些相对简单、重复性高的任务，如批准操作、数据压缩、记忆的重刷、调用 [[MCP 调用]] 相关的技能、进行 Session 搜索、处理视觉信息或网页内容等，则可以交给副模型。

配置上的关键在于，用户可以在 `config.yaml` 文件中明确指定主模型和副模型。为了实现 [[性能与成本平衡]]，通常建议将副模型设置为高性价比的模型，例如 Google 的 [[Gemini 1.5 Flash]]。这样做的好处是，即使副模型需要频繁调用，其产生的 [[Token 消耗]] 和 [[运营成本]] 也会显著降低，从而整体上提升了 Agent 的经济性，同时不牺牲处理复杂任务的能力。这种架构设计直接响应了用户关于成本的反馈，是 [[Token 节省策略]] 的重要体现。

## 应用场景

用户反馈驱动的 [[主副模型架构]] 在多种场景下都能发挥重要作用：

1.  **成本敏感型应用：** 对于需要大规模部署或长时间运行的 Agent 应用，如智能客服、内容生成助手、自动化工作流等，用户对 [[Token 消耗]] 的敏感度极高。通过将辅助任务分配给低成本副模型，可以显著降低整体的运营成本，使得 Agent 的部署更加经济可行。
2.  **交互体验优化：** 用户反馈也可能集中在交互的流畅性和响应速度上。通过将一些非核心的、可以异步处理的任务（如数据预处理、初步信息筛选）交给副模型，可以确保主模型能够更快地响应用户的核心指令，提升整体的 [[交互体验]]。
3.  **特定任务的效率提升：** 对于某些专门设计的辅助任务，例如批量处理、数据压缩、格式转换等，使用专门优化过的、成本较低的模型可能比通用大模型更高效。用户反馈可以帮助识别出这些可以被“外包”给副模型的任务类型。
4.  **多模态处理的成本控制：** 当 Agent 需要处理视觉或网页信息时，这些任务往往消耗较多 [[Token]]。通过将这些 [[视觉任务]] 和 [[网页处理]] 分配给专门的、成本较低的副模型，可以有效控制多模态处理的 [[Token 消耗]]。

总而言之，用户反馈是识别和优化 Agent 成本与性能的关键驱动力，使得 [[Hermes Agent]] 能够更好地适应实际应用需求。
