## 概念定义

简单任务是指在 [[AI Agent]] 的工作流程中，那些计算量较小、逻辑相对直接、对模型能力要求不高的任务。这类任务通常不涉及复杂的推理、长篇幅的内容生成、深度分析或需要高度创造性的输出。在 [[Hermes Agent]] 的 [[主副模型架构]] 中，简单任务是分配给 [[低成本副模型]]（如 [[Gemini 1.5 Flash]]）的主要处理对象。这种任务分工策略的核心在于实现 [[Token 优化配置]]，通过将大量低成本的简单任务转移给性价比更高的模型，从而显著降低整体的 [[Token 消耗]] 和 [[运营成本]]。例如，信息检索、数据格式转换、简单的文本摘要、事实核查等都可以被归类为简单任务。通过有效地识别和分配简单任务，Agent 能够更高效地利用计算资源，将昂贵的主模型（Main Model）留给处理 [[复杂逻辑]] 和 [[核心决策]] 等更具挑战性的任务，从而在 [[性能与成本平衡]] 方面达到最佳状态。

## 技术细节

在 [[Hermes Agent]] 的 [[Token 优化配置]] 策略中，简单任务的处理流程是这样的：当 Agent 接收到一个用户请求时，系统会首先评估任务的复杂度。如果任务被判定为“简单”，它会被路由到预设的副模型（Auxiliary Model）。这些副模型通常是经过优化、成本较低的模型，例如 [[Gemini 1.5 Flash]]。副模型处理完简单任务后，其结果会返回给主模型，或者直接呈现给用户。这种机制依赖于 Agent 内部的智能任务调度和分类能力。系统需要具备一定的能力来区分任务的复杂程度，这可能涉及到对输入提示的分析、对预期输出长度的预估，以及对所需计算资源的初步判断。通过这种方式，Agent 能够实现“任务分工”，将有限的计算资源和高成本的 Token 用于处理真正需要强大模型能力的部分，而将大量重复性、低复杂度的工作交给更经济的副模型来完成。

## 应用场景

简单任务的识别和分配在多种场景下都能带来显著的效益。例如，在 [[Hermes Agent]] 的 [[交互界面升级]] 中，当用户进行简单的信息查询、文本格式调整（如将普通文本转换为 Markdown 格式）、或者进行一些基础的文本润色时，这些都可以被视为简单任务。通过将这些任务分配给低成本副模型，可以确保用户界面的响应速度，同时避免不必要的 Token 消耗。另一个应用场景是 [[云端模型集成]]，当用户使用 [[Ollama]] 集成了 [[Minimax M2.7]] 等云端模型时，如果用户提出的请求是简单的指令执行或信息提取，同样可以优先考虑使用副模型来处理，以最大化免费模型额度的利用率。此外，在需要大规模部署 Agent 的生产环境中，[[Token 节省策略]] 是至关重要的，将简单任务有效分流是降低 [[运营开支]] 的关键手段之一。
