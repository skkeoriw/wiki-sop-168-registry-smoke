## 概念定义

预算控制是 Claude Code 动态工作流（Dynamic Workflows）中的一项关键功能，旨在管理和限制大型语言模型（LLM）在执行复杂任务时的 Token 消耗。在处理海量数据、进行深度代码分析或多模型协作时，Token 消耗可能迅速累积，导致成本失控或超出预期。预算控制允许用户为特定的任务或整个工作流设定一个明确的 Token 上限（例如，200 万 Token）。一旦达到或接近此上限，工作流将采取预设的行动，如暂停执行、发出警告、切换到更经济的模型，或自动终止任务。这项机制确保了分析的深度与成本效益之间的平衡，使得用户能够在可接受的资源范围内获得高质量的分析结果，避免不必要的资源浪费和 [[资源失控]]。通过精细化的预算管理，可以显著提升 [[Claude Code Dynamic Workflows 高级用法与实践简报]] 的整体 [[性价比]]。

## 技术细节

在 Claude Code 的动态工作流中，预算控制通常通过以下方式实现：

*   **Token 预算设置：** 用户可以在工作流的配置或脚本中明确指定一个 Token 预算值。这个值可以针对单个 [[智能体]]、特定任务阶段，或整个工作流。
*   **实时监控：** 工作流引擎会实时跟踪每个模型调用和 [[智能体]] 任务的 Token 消耗。
*   **触发机制：** 当累计 Token 消耗达到预设预算的某个百分比（例如 90%）或完全达到预算上限时，会触发预设的响应机制。
*   **响应策略：** 预设的响应策略可能包括：
    *   **警告通知：** 向用户发送通知，告知预算即将耗尽。
    *   **模型降级：** 自动切换到更低成本但性能稍逊的模型（如从 [[Opus]] 切换到 [[Sonnet]] 或 [[Haiku]]）。
    *   **任务暂停/终止：** 暂停当前任务或整个工作流，等待用户进一步指示，或直接终止任务以防止超额消耗。
    *   **优化建议：** 提供关于如何减少 Token 消耗的建议，例如精简输入数据、优化提示词等。
*   **与 [[CodeGraph MCP]] 集成：** 在集成 [[CodeGraph MCP]] 进行代码导航时，预算控制尤为重要，因为深度代码跳转和关联分析可能涉及大量的 Token 消耗。通过设置合理的预算，可以确保 [[代码导航]] 过程既深入又经济。

## 应用场景

*   **大规模代码库分析：** 在对包含数百万行代码的 [[代码库]] 进行分析时，设置 Token 预算可以防止分析过程因消耗过多 Token 而中断或变得过于昂贵。
*   **多模型协作与 [[对抗验证]]：** 当工作流中同时调用多个模型（如 [[Opus]]、[[Sonnet]]、[[Codex CLI]]）进行交叉比对时，预算控制可以确保整体成本在可控范围内，并帮助评估不同模型在 [[效率对比]] 上的表现。例如，在 [[Codex CLI 交叉验证代码审查]] 场景中，可以为整个审查流程设定一个总预算，并监控每个模型的消耗。
*   **[[GitHub Issue 分析]]：** 分析大量的 GitHub Issue 和相关的代码提交，以识别 [[核心 Bug]]。预算控制可以确保在有限的资源下，尽可能多地覆盖重要的 Issue。
*   **[[数据拉取]] 与初步分析：** 在处理来自外部源的大量数据时，预算控制可以限制初步数据抓取和清洗阶段的 Token 消耗，为后续更深入的分析保留预算。
