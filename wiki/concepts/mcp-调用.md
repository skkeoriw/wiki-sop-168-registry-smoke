## 概念定义

MCP 调用（MCP Call）是 [[Hermes Agent]] 框架中用于处理特定类型辅助任务的一种机制。在 [[主副模型架构]] 中，Hermes Agent 允许用户配置一个主模型来处理复杂逻辑和核心决策，同时配置一个或多个副模型（辅助任务模型）来处理相对简单或重复性的任务。MCP 调用就属于这些辅助任务的范畴，它通常与批准、压缩、重刷记忆、Session 搜索、视觉及网页处理等任务一同被分配给副模型。这种设计旨在通过将不同复杂度的任务分配给最适合的模型，从而实现 [[Token 优化配置]]，显著降低 [[Token 消耗]]，并达到 [[性能与成本平衡]] 的最佳效果。

## 技术细节

MCP 调用本身并非一个独立的模型或 API，而是 Hermes Agent 内部的一种任务调度和执行方式。当用户触发需要执行 MCP 调用相关的操作时，Hermes Agent 会将该请求路由给预先配置的副模型。这些副模型通常是 [[高性价比模型]]，例如 [[Gemini 1.5 Flash]]，因为它们在处理这类辅助任务时能够提供良好的性能且成本较低。接口设置中，Hermes Agent 的连接地址通常指向 `http://localhost:8642/v1`，这表明其可能通过本地服务或特定端口与模型进行通信。MCP 调用的具体实现细节可能依赖于 Hermes Agent 的内部设计以及所集成的第三方模型 API。其核心在于将特定类型的“技能”或“调用”抽象出来，并灵活地分配给不同的模型实例。

## 应用场景

MCP 调用的主要应用场景在于优化 [[Hermes Agent]] 的整体运行效率和成本控制。具体而言，它被用于处理以下类型的辅助任务：

*   **批准（Approval）**: 在某些工作流中，可能需要模型对某个操作进行确认或批准。
*   **压缩（Compression）**: 对数据或信息进行压缩，以减少存储或传输的开销。
*   **重刷记忆（Memory Refresh）**: 更新或重置 Agent 的内部记忆状态。
*   **MCP 调用（MCP Call）**: 这是指代一类特定的辅助调用，具体功能可能根据 Agent 的设计而异，但通常是执行某个预定义的操作。
*   **Session 搜索（Session Search）**: 在用户会话历史中进行搜索，以检索相关信息。
*   **视觉及网页处理（Vision and Web Processing）**: 处理图像信息或从网页中提取数据。

通过将这些任务分配给成本更低的副模型，用户可以显著节省 [[Token 消耗]]，尤其是在处理大量请求或复杂交互时，这种策略能够有效降低 [[运营成本]]。例如，在 [[Open WebUI]] 中，用户可以通过配置来指定哪些任务由副模型处理，从而实现精细化的成本管理。
