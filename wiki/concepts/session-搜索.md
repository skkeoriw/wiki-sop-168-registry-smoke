## 概念定义

Session 搜索（Session Search）是 Hermes Agent 框架中一项重要的辅助任务，它被归类为由副模型（辅助任务模型）处理的子任务之一。在 Hermes Agent 的主副模型架构设计中，主模型负责处理复杂逻辑和核心决策，而副模型则承担一系列相对简单但关键的辅助功能，以实现 Token 的深度节省和整体效率的提升。Session 搜索正是这些辅助功能中的一个具体体现。其核心目的是在 Agent 的运行过程中，能够高效地检索和管理与当前会话（Session）相关的历史信息、上下文数据或特定状态，从而为 Agent 的后续决策和响应提供必要的数据支持，同时避免不必要的 Token 消耗。通过将 Session 搜索这类任务分配给高性价比的副模型，Hermes Agent 能够显著降低整体的 Token 使用量，实现性能与成本的最佳平衡。

## 技术细节

Session 搜索作为副模型处理的任务之一，其技术实现通常与 Agent 的内部状态管理和记忆机制紧密相关。当 Agent 需要访问或检索特定会话的历史信息时，例如用户之前的提问、Agent 的回答、中间的思考过程，或者与特定任务相关的上下文数据，就会触发 Session 搜索。副模型会根据预设的规则或算法，从 Agent 的记忆库或当前会话的上下文中，快速定位并提取所需信息。这种机制的设计是为了避免将所有历史信息都打包发送给主模型进行处理，从而节省 Token。

在 Hermes Agent 的配置中，用户可以指定一个高性价比的模型（如 Gemini 1.5 Flash）作为副模型来处理 Session 搜索等辅助任务。这意味着，当需要进行 Session 搜索时，请求会被路由到这个指定的副模型。副模型会执行其内部的搜索逻辑，找到相关信息后，再将结果返回给主模型，或者直接用于 Agent 的后续流程。这种任务分工和模型选择的灵活性，是 Hermes Agent 实现 Token 节省和成本优化的关键技术手段。

## 应用场景

Session 搜索在 Hermes Agent 的多种应用场景中都发挥着重要作用：

1.  **多轮对话管理：** 在进行长时间的多轮对话时，Agent 需要能够回忆起之前的对话内容，以便提供连贯且相关的回应。Session 搜索可以帮助 Agent 快速检索历史对话片段，理解上下文，避免重复提问或给出不相关的答案。
2.  **任务状态追踪：** 对于需要执行一系列步骤的复杂任务，Agent 需要追踪任务的当前状态、已完成的子任务以及待执行的任务。Session 搜索可以帮助 Agent 检索和更新这些任务状态信息。
3.  **个性化响应：** 如果 Agent 需要根据用户的历史偏好或行为模式提供个性化服务，Session 搜索可以帮助 Agent 检索用户的相关信息，从而生成更符合用户需求的响应。
4.  **信息检索与整合：** 在处理需要整合来自不同来源的信息时，Session 搜索可以帮助 Agent 快速定位和提取与当前任务相关的历史信息片段，为信息整合提供基础。

通过将 Session 搜索分配给副模型，Hermes Agent 能够更高效地处理这些场景，同时显著降低 Token 的消耗，为用户提供更经济、更流畅的交互体验。
