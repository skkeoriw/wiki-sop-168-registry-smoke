## 主副模型架构

主副模型架构（Main/Auxiliary Model Architecture）是一种在 AI Agent 系统中用于优化成本和性能的策略。该架构的核心思想是将 AI Agent 的任务进行划分，并根据任务的复杂度和对成本的要求，将其分配给不同类型的语言模型。通常，一个“主模型”（Main Model）负责处理复杂、需要深度理解和推理的任务，而一个或多个“副模型”（Auxiliary Model）则负责处理相对简单、重复性高或对成本敏感的任务。

这种架构的优势在于能够实现成本与性能的平衡。例如，对于一些简单的指令执行、信息检索或数据预处理等任务，可以选用成本较低、响应速度快的副模型，如 [[Gemini 1.5 Flash]] 或其他经过优化的轻量级模型。而对于需要进行复杂逻辑判断、创意生成或深度分析的任务，则会调用更强大但成本更高的主模型。通过这种精细化的任务分工，可以显著降低整体的 [[Token 消耗]]，从而减少 [[运营成本]]，尤其是在 [[生产环境部署]] 中，这种策略能够带来显著的经济效益。

### 技术细节

在实现主副模型架构时，关键在于如何有效地识别和路由任务。这通常涉及到：

*   **任务分类器：** 一个能够根据任务的输入、指令和上下文，判断其复杂度和所需模型类型的模块。
*   **模型路由机制：** 根据任务分类器的判断结果，将请求准确地发送到指定的主模型或副模型。
*   **模型能力评估：** 对不同模型的能力、成本和响应时间进行持续的评估和更新，以便做出最优的路由决策。
*   **结果整合：** 当任务被分解并由不同模型处理后，需要一个机制将各部分结果有效整合，形成最终的输出。

### 应用场景

主副模型架构在多种 AI Agent 应用中具有广泛的应用前景：

*   **智能客服：** 将简单的问答和信息查询分配给副模型，将复杂的用户问题和情感分析交给主模型。
*   **内容生成：** 对于批量生成相似内容（如产品描述）的任务，可以使用副模型；而对于需要高度原创性和创造性的内容（如小说创作），则调用主模型。
*   **数据分析：** 简单的统计和数据清洗任务可由副模型处理，而复杂的模式识别和预测模型则由主模型负责。
*   **[[Hermes Agent]] 的 Token 优化：** 如 [[Hermes-Agent-高级玩法与配置指南-稳定性-交互优化与-Token-节.md]] 中所述，Hermes Agent 引入主副模型机制，将简单任务分配给低成本副模型，以大幅降低 [[运营成本]]。

这种架构不仅能有效控制 [[Token 消耗]]，还能提升系统的整体响应速度和用户体验，是构建高效、经济的 AI Agent 系统的关键技术之一。
