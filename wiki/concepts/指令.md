## 概念定义

指令（Command）在 [[Hermes Agent]] 的语境下，是指用户向 Agent 发出的用于执行特定操作或配置其行为的命令或请求。这些指令可以是简单的文本输入，也可以是包含特定参数的结构化命令。它们是 Agent 能够理解并响应用户意图的关键。指令的本质是传递信息和意图，驱动 Agent 的执行流程。例如，用户可以通过指令来启动一个任务、调整 Agent 的参数、连接外部服务，或者要求 Agent 执行特定的计算或分析。指令的清晰度和准确性直接影响 Agent 的执行效率和结果的质量。在 [[Hermes Agent]] 的高级用法中，指令的设计和使用变得尤为重要，它涉及到如何有效地利用 [[Ollama]] 进行部署，如何通过 [[Open WebUI]] 提升交互体验，以及如何通过 [[主副模型架构]] 来优化 [[Token 消耗]]。指令的灵活性和强大功能是实现 Agent 高级玩法和精细化配置的基础。

## 技术细节

指令在 [[Hermes Agent]] 中扮演着核心的角色，其技术实现体现在多个方面：

*   **Ollama 集成与部署指令：** [[Ollama]] 原生内置了 Hermes Agent，用户只需一条简单的指令即可完成本地环境与云端能力的连接。这使得“零成本上手”成为可能，用户可以利用 [[Ollama]] 提供的免费模型额度（如 [[Minimax M2.7]] 云端版），实现“傻瓜式”配置，无需占用本地硬件资源。
*   **Open WebUI 配置指令：** 为了提升交互体验，[[Open WebUI]] 提供了类似 ChatGPT 的功能，包括侧边栏历史记录、流式输出、语音发音、代码块运行等。配置 [[Open WebUI]] 需要在 `config.yaml` 中启用 API 服务 (`enable_api: true`) 并设置 API 访问密码。用户通过在 [[Open WebUI]] 接口设置中连接 `http://localhost:8642/v1` 来发送指令，实现跨端访问，支持文件上传、截图引用及网页引用。
*   **主副模型架构指令：** 针对 [[Token 消耗]] 过高的问题，Hermes Agent 支持配置不同的模型处理不同复杂度的任务。用户可以通过指令来指定主模型（处理复杂逻辑、核心决策）和副模型（处理批准、压缩、重刷记忆、MCP 调用等辅助任务）。这种 [[任务分工]] 的指令配置，能够显著降低 [[Token 消耗]]，实现 [[性能与成本平衡]]。
*   **API 服务指令：** 启用 [[API 服务]] 是实现 [[Open WebUI]] 集成和远程访问的关键。通过设置 [[API 访问密码]]，可以确保通信的安全性。这些指令的配置直接影响 Agent 的可访问性和功能扩展性。

## 应用场景

指令在 [[Hermes Agent]] 的应用场景中无处不在，主要体现在以下几个方面：

*   **快速部署与体验：** 通过简单的部署指令，用户可以快速启动 [[Hermes Agent]]，并利用 [[Ollama]] 的免费模型额度进行体验，无需复杂的环境搭建和硬件投入。这使得 [[免费模型利用]] 和 [[零成本上手]] 成为可能。
*   **增强交互体验：** 用户可以通过指令与 [[Open WebUI]] 进行交互，享受更流畅的对话体验，包括 [[流式输出]]、[[语音发音]]、[[代码块运行]] 等。同时，[[跨端访问]] 的能力也让用户可以在手机浏览器上直接与 Agent 互动，实现 [[文件上传]]、[[截图引用]] 和 [[网页引用]]。
*   **精细化任务处理与成本控制：** 通过指令配置 [[主副模型架构]]，用户可以根据任务的复杂度选择合适的模型，例如将 [[复杂逻辑]] 和 [[核心决策]] 交给主模型，将 [[批准]]、[[压缩]] 等 [[辅助任务]] 交给低成本的副模型。这有助于实现 [[Token 节省策略]]，降低 [[运营成本]]，尤其适用于 [[长线任务]] 和 [[生产环境部署]]。
*   **集成与扩展：** 指令是连接 [[Hermes Agent]] 与其他服务和工具的桥梁。通过配置 [[API 服务]]，可以实现与其他应用程序的集成，扩展 Agent 的功能，例如处理 [[视觉任务]] 或 [[网页处理]] 等。
