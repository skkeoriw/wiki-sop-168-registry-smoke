## 概念定义

压缩（Compression）在 Hermes Agent 的语境下，并非指传统意义上的文件或数据压缩，而是指一种策略性的模型资源分配方式，旨在通过优化 Token 的使用效率来降低整体运营成本。具体而言，它指的是将一些相对简单、不那么核心的任务，例如批准、重刷记忆、MCP 调用、Session 搜索、视觉及网页处理等辅助性任务，分配给成本更低、性价比更高的模型（如 Gemini 1.5 Flash），而将处理复杂逻辑和核心决策等关键任务留给主模型。这种主副模型架构的设计，能够有效避免将所有任务都交给昂贵的主模型处理，从而显著减少 Token 的消耗，实现性能与成本的最佳平衡。通过这种“压缩”策略，用户可以更经济高效地利用 AI Agent 的能力，尤其是在处理大量请求或长线任务时，其成本优势尤为明显。

## 技术细节

压缩策略的核心在于 Hermes Agent 的 [[主副模型架构]]。用户可以根据任务的复杂度和对成本的要求，灵活配置主模型和副模型。

*   **主模型：** 负责处理需要深度理解、复杂推理和关键决策的任务。通常会选择性能更强但成本也相对较高的模型。
*   **副模型（辅助任务）：** 负责处理如批准、压缩、重刷记忆、MCP 调用、Session 搜索、视觉及网页处理等辅助性任务。为了实现成本效益，建议将这些任务指定给高性价比模型，例如 Google 的 Gemini 1.5 Flash。

这种分工明确的架构，使得 Agent 能够根据任务的性质动态调用最合适的模型，从而在保证功能完整性的前提下，最大程度地节省 [[Token 消耗]]。配置通常在 Agent 的接口设置中进行，例如通过 Open WebUI 接口设置连接到本地运行的模型服务。

## 应用场景

压缩策略的应用场景非常广泛，尤其是在以下方面：

*   **成本控制：** 对于预算有限的用户或需要大规模部署 Agent 的场景，通过使用低成本副模型处理大量辅助任务，可以显著降低 [[运营成本]]。
*   **性能优化：** 将简单任务交给副模型处理，可以释放主模型的计算资源，使其能够更专注于复杂任务，从而提升整体响应速度和处理效率。
*   **长线任务处理：** 在需要 Agent 持续运行并处理大量信息的长线任务中，压缩策略能够有效控制 Token 消耗，避免因成本过高而中断任务。
*   **多模态任务处理：** 对于涉及视觉和网页处理等辅助性多模态任务，将其分配给专门优化的副模型，可以提高处理效率并降低成本。

通过这种精细化的模型管理，Hermes Agent 能够更好地满足用户在性能、成本和效率方面的多重需求。
