## 概念定义

提示词（Prompting）是当前软件开发范式 [[软件 3.0]] 的核心组成部分。在这一新范式下，编程不再是编写一系列明确的、由开发者预定义的指令（如 [[软件 1.0]] 中的传统源代码），而是通过精心设计的文本输入，即提示词，来引导和控制大型语言模型（LLM）这一新型“解释器”的行为。提示词充当了与 LLM 沟通的接口，开发者通过提供上下文（Context）、指令、示例或约束条件，来“编程” LLM，使其执行特定的任务、生成期望的输出，或者扮演特定的角色。一个好的提示词能够有效地激发 LLM 的能力，使其在复杂任务中表现出惊人的准确性和创造力，从而实现效率的百倍提升。提示词工程（Prompt Engineering）因此成为 [[代理工程]] 的关键技能，它要求开发者不仅要理解 LLM 的工作原理，还要具备将人类意图转化为 LLM 能够理解和执行的语言指令的能力。

## 技术细节

提示词的设计涉及多个维度，包括：

*   **指令（Instruction）：** 明确告知 LLM 需要做什么，例如“写一篇关于…”、“总结以下文本…”、“生成一段 Python 代码来…”等。
*   **上下文（Context）：** 提供 LLM 进行推理和生成所需的相关信息，可以是先前的对话、文档片段、数据点等。
*   **示例（Examples）：** 通过提供输入-输出对的示例，让 LLM 学习期望的格式、风格或行为模式，这是一种“少样本学习”（Few-shot Learning）或“零样本学习”（Zero-shot Learning）的体现。
*   **约束（Constraints）：** 设定输出的限制条件，例如字数限制、语言风格要求、禁止包含特定内容等。
*   **角色扮演（Role-playing）：** 要求 LLM 扮演某个特定的角色，如“你是一位资深软件工程师…”、“你是一位历史学家…”等，以引导其输出符合该角色的专业性和视角。

提示词的有效性直接影响 LLM 的输出质量和任务完成度。随着模型能力的增强，提示词的复杂性和精细度也在不断提升，从简单的指令到复杂的、多步骤的推理链。

## 应用场景

提示词的应用场景极其广泛，几乎涵盖了所有需要与 LLM 交互的领域：

*   **代码生成与辅助：** 开发者可以通过提示词让 LLM 生成代码片段、解释代码、调试代码，甚至完成整个函数或模块。
*   **内容创作：** 文章撰写、故事创作、诗歌生成、营销文案编写等。
*   **信息提取与总结：** 从大量文本中提取关键信息、生成摘要、回答问题。
*   **数据分析与洞察：** 解释数据、生成报告、发现趋势。
*   **对话式 AI：** 构建智能客服、虚拟助手、聊天机器人。
*   **教育与培训：** 生成学习材料、提供个性化辅导、模拟面试。
*   **软件开发流程自动化：** 通过提示词驱动的代理来执行任务，如环境配置、部署脚本生成等。

在 [[Andrej Karpathy：从氛围编程到代理工程的技术简报]] 中，提示词被视为软件 3.0 的核心，标志着编程范式的根本性转变，从编写显式规则转向通过与 LLM 的交互来驱动软件的构建和运行。
