## 概念定义

网页处理（Web Processing）是 [[Hermes Agent]] 框架中一个重要的辅助功能，它指的是 Agent 能够理解、解析和操作网页内容的能力。在 [[Hermes Agent]] 的主副模型架构中，网页处理任务通常被归类为辅助任务，并分配给副模型（如 [[Gemini 1.5 Flash]] 等高性价比模型）来执行。这种分工策略旨在通过将相对简单或重复性的任务交给成本更低的副模型，从而显著降低整体的 [[Token 消耗]]，实现 [[性能与成本平衡]]。网页处理功能使得 Agent 能够从互联网上获取信息、理解网页结构、提取关键数据，甚至模拟用户在网页上的交互行为，极大地扩展了 Agent 的应用范围和智能化水平。

## 技术细节

在 [[Hermes Agent]] 的主副模型架构下，网页处理任务被明确地定义为一种辅助任务。这意味着当 Agent 需要执行诸如浏览网页、提取信息、分析页面结构等操作时，这些任务会被路由到配置的副模型。副模型通常是经过优化以处理特定类型任务的模型，例如 [[Gemini 1.5 Flash]]，其特点是计算效率高且成本较低。这种设计允许主模型（负责复杂逻辑和核心决策）专注于更具挑战性的任务，而副模型则高效地处理如网页处理、批准、压缩、重刷记忆、MCP 调用、Session 搜索、视觉任务等辅助性工作。通过这种方式，Hermes Agent 能够更精细地管理计算资源，避免不必要的 [[Token 消耗]]，尤其是在处理大量网页信息时，能够显著提高 [[Token 优化配置]] 的效果。

## 应用场景

网页处理功能在多种场景下都能发挥重要作用。例如，在信息聚合类应用中，Agent 可以通过网页处理能力抓取和分析来自不同新闻网站、博客或论坛的信息，为用户提供全面的资讯摘要。在研究和数据分析领域，Agent 可以自动爬取特定网站的数据，进行初步的整理和提取，为后续的深度分析奠定基础。此外，在需要用户与网页进行交互的场景下，如填写表单、点击链接或执行特定操作，网页处理能力也至关重要。通过将这类任务分配给高性价比的副模型，[[Hermes Agent]] 能够以更低的成本提供更广泛的服务，满足用户在信息获取、数据处理和自动化交互等方面的多样化需求，是实现 [[Token 节省策略]] 和提升 [[交互体验]] 的关键组成部分。
