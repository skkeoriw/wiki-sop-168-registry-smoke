## 概念定义

独立智能体（Independent Agent）是动态工作流（Dynamic Workflow）执行过程中的一个核心概念，特指在多智能体协作模式下，为处理工作流中的特定任务点（Task Point）而独立创建和运行的智能体实例。在“并行分析”阶段，系统会为每个需要处理的任务（例如，GitHub 中的一个 Issue 或代码库中的一个文件）派生出一个或多个独立智能体。这些智能体各自拥有独立的执行上下文和能力，可以并行地执行分析、验证或生成等操作，而无需等待其他智能体的完成。这种设计极大地提升了工作流的效率，尤其是在处理海量数据或复杂任务时。例如，当需要处理 50 个 GitHub Issue 时，系统可以同时启动数十个独立智能体实例，每个实例负责分析一个或一组 Issue，从而实现大规模并发处理。独立智能体的存在是实现高效、可扩展工作流的关键，它们使得复杂的任务能够被分解成更小、更易于管理的单元，并能被并行化处理，最终由一个核心智能体进行汇总和决策。

## 技术细节

独立智能体的创建和管理是动态工作流引擎的核心功能之一。在“并行分析”阶段，工作流引擎会根据预设的逻辑和任务分配策略，为每个任务点动态地派生出独立的智能体实例。这些智能体可以被配置为使用不同的模型（如 [[Opus]]、[[Sonnet]] 或 [[Haiku]]），或者调用特定的 [[外部工具]]（如 [[CodeGraph MCP]] 或 [[Codex CLI]]），以执行特定的分析或操作。

*   **派生机制：** 工作流脚本定义了派生独立智能体的条件和参数。当满足条件时，引擎会实例化一个或多个智能体。
*   **并发执行：** 独立智能体支持大规模并发执行。系统能够同时启动数十个甚至上百个智能体实例，以并行处理大量任务。
*   **资源隔离：** 每个独立智能体通常拥有独立的执行环境和资源配额（如 Token 预算），以防止相互干扰并便于成本控制。
*   **状态管理：** 工作流引擎负责跟踪每个独立智能体的执行状态，包括启动、运行中、完成、失败等。
*   **结果汇聚：** 完成的独立智能体将其分析结果或中间产出提交给工作流引擎，通常会由一个“核心智能体”进行汇总和整合，形成最终的输出。
*   **对抗验证：** 为了提高结果的准确性，可以派生多个独立智能体，使用不同的模型或方法对同一任务进行分析，然后通过“对抗验证”阶段进行交叉比对，以识别和纠正潜在的错误。

## 应用场景

独立智能体的应用场景非常广泛，尤其是在需要处理大量数据、执行复杂分析或进行大规模验证的任务中。

*   **大规模 GitHub Issue 分析与验证：** 针对拥有大量待处理 Issue 的开源项目，可以派生大量独立智能体，并行分析每个 Issue，自动筛选出 T10 级别的核心 Bug，并根据“崩溃数据”和“数据丢失”等信号进行优先级排序。
*   **代码库导航与分析：** 集成 [[CodeGraph MCP]] 等工具，独立智能体可以深入代码库进行关联分析和代码跳转，理解复杂的代码逻辑。
*   **多模型交叉验证：** 在代码审查场景中，可以派生使用不同模型（包括非 Claude 系列模型，如 [[Codex CLI]]）的独立智能体，对同一段代码进行交叉验证，利用不同模型的互补优势，发现潜在的 Bug。
*   **数据处理与转换：** 对于需要对海量数据进行清洗、转换或特征提取的任务，可以为每个数据块或数据点派生独立智能体，并行完成处理。
*   **自动化测试：** 在软件测试流程中，可以为不同的测试用例或测试场景派生独立智能体，并行执行测试，并收集测试结果。
