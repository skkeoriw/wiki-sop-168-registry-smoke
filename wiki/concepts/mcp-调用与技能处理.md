## 概念定义

MCP 调用与技能处理（MCP Call and Skill Handling）是一种将 AI Agent 的特定功能模块化并分配给不同计算资源的策略。在 [[Hermes Agent]] 的高级配置中，它被归类为“副模型（Auxiliary Tasks）”的推荐配置项之一。这意味着，一些非核心但必要的任务，例如用户请求的审批（Approval）、数据压缩（Compression）、记忆刷新（Memory Refresh）、MCP 调用（MCP Call）以及处理视觉与网页信息（Vision/Web Tasks），可以被专门指派给一个或多个副模型来执行。这种分工的目的是为了优化整体 Agent 的性能和成本效益。通过将这些辅助性任务从主模型中剥离，可以减轻主模型的负担，使其更专注于处理核心的、复杂的逻辑和决策。同时，副模型可以根据任务的特性选择更经济高效的模型，例如 [[Gemini 1.5 Flash]]，从而在保证功能正常运行的前提下，显著降低 [[Token 消耗]] 和整体 [[运营成本]]。这种架构设计体现了 [[主副模型架构]] 的核心理念，即根据任务的复杂度和重要性进行资源分配，实现 [[性能与成本平衡]]。

## 技术细节

在 [[Hermes Agent]] 的配置文件中，用户可以明确指定哪些任务应被视为“副模型”任务。上述提到的“审批”、“压缩”、“记忆刷新”、“MCP 调用与技能处理”、“视觉与网页任务”均是推荐被细化至副模型的任务类型。这意味着，当 Agent 需要执行这些功能时，它不会直接调用主模型，而是将请求路由到预先配置好的副模型实例。这些副模型可以选用成本较低、响应速度快的模型，例如 Gemini 1.5 Flash，以满足这些辅助性任务的需求。配置的关键在于区分主模型和副模型，并将非核心的辅助性任务切换至廉价 API 或低成本模型。例如，对于简单的 [[MCP 调用]]，使用一个专门为此设计的副模型比让主模型处理效率更高且成本更低。此外，副模型还可以处理一些与 [[API 服务]] 相关的交互，例如通过 [[API 密钥]] 进行身份验证，并兼容 [[OpenAPI 接口]]。这种精细化的任务分工有助于实现 [[Token 节省策略]]，尤其是在 [[生产环境部署]] 中，能够大幅降低 [[运营开支]]。

## 应用场景

MCP 调用与技能处理的核心应用场景在于优化 AI Agent 的整体效率和成本控制，尤其是在需要处理大量辅助性任务的复杂应用中。

1.  **成本效益最大化：** 在 [[生产环境部署]] 中，将诸如数据压缩、记忆刷新、简单的 [[MCP 调用]] 等任务分配给成本较低的副模型，可以显著降低 [[Token 消耗]] 和整体 [[运营成本]]。这对于需要处理大量用户请求的 Agent 尤为重要。
2.  **提升主模型性能：** 通过将辅助性任务（如 [[视觉任务]] 和 [[网页处理]]）交给副模型，可以减轻主模型的负担，使其能够更专注于处理核心的、复杂的逻辑和决策，从而提升 Agent 的响应速度和处理能力。
3.  **模块化与可维护性：** 将不同的技能和任务模块化并分配给副模型，使得 Agent 的架构更加清晰，易于维护和升级。例如，如果某个副模型的性能需要提升，可以单独对其进行优化或更换，而不会影响到主模型的运行。
4.  **支持多样化任务：** Agent 可以通过配置不同的副模型来支持各种类型的任务，例如处理用户上传的文件、进行 [[截图引用]]、解析 [[网页引用]]，甚至执行一些特定的 [[代码块运行]]。
5.  **远程访问与跨端协作：** 当用户需要在户外通过手机访问家庭 Agent 时，可以将 [[Open WebUI]] 的端口暴露出来，并结合 [[内网穿透]] 技术实现远程访问。此时，Agent 的副模型可以高效地处理用户请求，并快速响应。

总而言之，MCP 调用与技能处理是一种精细化管理 AI Agent 资源和能力的策略，旨在实现高性能、低成本和高可维护性的目标。
