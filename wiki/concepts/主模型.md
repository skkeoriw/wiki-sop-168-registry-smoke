# 主模型

在 [[Hermes Agent]] 的架构中，**主模型**扮演着至关重要的角色，它被设计用来处理复杂、核心的逻辑推理和决策制定。与副模型（辅助任务模型）相配合，主副模型架构旨在实现 [[Token 优化配置]]，显著降低 [[Token 消耗]]，从而在保证 Agent 强大功能的同时，优化运营成本。

## 技术细节

主模型是 Agent 的“大脑”，负责理解和执行那些需要深度思考、多步骤推理或涉及关键判断的任务。例如，当 Agent 需要进行复杂的规划、分析用户意图的深层含义、或者在多个选项中做出最优选择时，这些任务通常会交由主模型来处理。其选择和配置直接影响着 Agent 的整体智能水平和处理复杂问题的能力。

在 [[Hermes Agent 高级玩法与配置指南：稳定性、交互优化与 Token 节省策略]] 中，明确指出主模型应承担“复杂逻辑”和“核心决策”的任务。这意味着，为了最大化 Agent 的效能，用户需要根据实际需求，为这个关键角色选择一个能力强大、推理精准的模型。

## 应用场景

主模型在以下场景中发挥核心作用：

*   **复杂任务处理：** 当 Agent 需要执行需要多步骤推理、逻辑分析或综合信息才能完成的任务时，主模型是首选。
*   **核心决策制定：** 在需要 Agent 根据上下文和目标做出关键性选择时，主模型的判断能力至关重要。
*   **高级指令理解：** 对于包含细微差别、隐含意义或多重条件的指令，主模型能够更准确地解析和执行。

通过将复杂任务分配给主模型，而将诸如批准、压缩、重刷记忆、MCP 调用、Session 搜索、视觉及网页处理等辅助性任务分配给成本效益更高的副模型（如 [[Gemini 1.5 Flash]]），可以实现 [[性能与成本平衡]]，确保 Agent 在提供强大功能的同时，保持经济高效的运行。这种精细化的任务分工是 [[Hermes Agent]] 实现高级功能和用户体验优化的关键策略之一。
