# 重刷记忆

重刷记忆（Re-memoization）是 [[Hermes Agent]] 在其 [[主副模型架构]] 中用于处理一系列辅助性任务的一种机制。在 [[Token 优化配置]] 策略中，为了有效控制 [[Token 消耗]] 和降低 [[运营成本]]，Hermes Agent 允许用户将不同复杂度的任务分配给不同的模型。其中，重刷记忆被归类为辅助任务，通常会分配给成本效益更高、性能更优的副模型，例如 [[Gemini 1.5 Flash]]。这种任务分工旨在让主模型专注于处理复杂逻辑和核心决策，而将诸如批准、压缩、重刷记忆、MCP 调用、Session 搜索、视觉及网页处理等辅助任务交给副模型，从而实现 [[性能与成本平衡]]。通过将重刷记忆等辅助任务交给更经济的模型处理，可以显著减少整体 Token 消耗，提升 Agent 的运行效率和经济性。

## 技术细节

在 Hermes Agent 的配置中，重刷记忆作为一种辅助任务，其具体实现和调用方式与 [[MCP 调用]]、[[Session 搜索]] 等其他辅助任务类似。当 Agent 需要执行重刷记忆操作时，它会将其委托给预先配置的副模型。这种委托机制依赖于 Agent 内部的任务调度和模型路由系统。用户可以通过 WebUI 接口设置，例如连接 `http://localhost:8642/v1`，来配置主副模型的选择和任务分配。选择高性价比的副模型来处理重刷记忆，意味着该模型需要具备一定的上下文理解和信息重组能力，但无需达到主模型处理复杂逻辑所需的深度。这种设计是 [[Token 节省策略]] 的重要组成部分，通过精细化任务分配来优化资源利用。

## 应用场景

重刷记忆在 Hermes Agent 的应用场景主要体现在需要对 Agent 的短期或长期记忆进行更新、优化或重组的场景。例如，在多轮对话中，如果 Agent 的记忆内容变得冗余或出现偏差，重刷记忆可以帮助 Agent 清理无效信息，重新组织关键上下文，从而提升后续对话的连贯性和准确性。此外，在处理需要回顾大量历史信息以做出决策的场景下，重刷记忆可以帮助 Agent 更高效地检索和利用相关信息。在 [[Hermes Agent 高级玩法与配置指南：稳定性、交互优化与 Token 节省策略]] 中，将重刷记忆等辅助任务分配给副模型，是实现 [[Token 消耗]] 优化的关键手段之一，尤其是在处理长线任务或需要频繁记忆更新的复杂交互时，这种策略能有效降低 [[运营开支]]。
