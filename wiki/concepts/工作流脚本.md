# 工作流脚本

工作流脚本是 Claude Code 动态工作流的核心组成部分，它是一种指令集，用于定义和管理一系列自动化任务的执行流程。通过工作流脚本，用户可以指导 Claude Code 按照预设的逻辑进行“规划-审计-执行”的生命周期管理。在最新版本的 Claude Code 中，激活动态工作流的统一指令为 `ultra code`。一旦该指令被输入，系统便会进入动态工作流界面，准备解析和执行脚本所定义的任务。

工作流脚本的设计旨在支持复杂的多智能体协作模式。一个典型的工作流会经历数据拉取阶段（通常不消耗智能体配额），用于获取执行任务所需的信息，如 GitHub Issue 或代码元数据。随后，进入并行分析阶段，系统会为每个独立的任务点（例如，每个 Issue）派生出独立的智能体实例，实现大规模并发处理。为了确保结果的准确性和可靠性，工作流还会包含对抗验证阶段，通过不同的智能体或模型对分析结果进行交叉比对。最终，一个核心智能体负责汇总所有反馈，生成最终的报告或输出。这种分阶段、多智能体协作的模式，极大地提升了处理复杂任务的效率和质量。

## 技术细节

工作流脚本的执行遵循严格的生命周期管理：

*   **规划 (Workflow Plan)：** 在实际执行前，系统会根据脚本生成详细的执行计划和流程图。这使得用户能够清晰地了解任务的分解、步骤以及预期结果，而无需立即消耗计算资源。
*   **审计 (Audit)：** 用户可以在执行前审查工作流脚本的每一个细节，包括具体的执行步骤、将要使用的模型（如 [[Opus]]、[[Sonnet]]、[[Haiku]] 等），以及预估的计算资源消耗（[[消耗]]）。这一环节是确保工作流符合预期、避免 [[资源失控]] 的关键。
*   **执行 (Execution)：** 一旦审计通过，工作流脚本便会驱动系统的执行引擎。Claude Code 支持大规模并发执行，这意味着系统可以同时启动数十个甚至上百个智能体实例来处理并行任务，例如同时处理大量 [[GitHub Issue 分析]]。

工作流脚本还支持 [[多模型协作]]，通过合理配置不同模型（如 [[旗舰模型]] 与性价比更高的模型）的组合，实现 [[互补优势]]，在保证分析深度和 [[准确性]] 的同时，优化 [[成本与性能优化]]。脚本的复用性也得到了提升，允许用户保存和分享有效的脚本配置，提高 [[效率对比]]。

## 应用场景

工作流脚本的应用场景广泛，尤其在需要处理海量数据和复杂逻辑的任务时，其优势尤为突出：

*   **大规模 GitHub Issue 分析与验证：** 对于拥有大量待处理 Issue 的开源项目，工作流脚本可以自动化筛选出关键的 [[核心 Bug]]（如 T10 级别 Bug），并根据“崩溃数据”和“数据丢失”等 [[优先级信号]] 进行排序。通过 [[并行分析]] 和 [[对抗验证]]，可以高效地完成对大量 Issue 的分析和验证。
*   **CodeGraph MCP 集成与代码导航：** 工作流脚本可以与 [[CodeGraph MCP]] 等工具集成，实现更深层次的代码库理解和 [[代码导航]]。例如，脚本可以驱动智能体分析代码的 [[关联分析]]，并生成 [[代码跳转]] 指令，极大地提升了开发者的代码审查和理解效率。
*   **系统级架构分析：** 复杂系统的架构分析可以通过工作流脚本进行自动化。脚本可以驱动智能体爬取 [[代码元数据]]，分析模块间的依赖关系，识别潜在的 [[潜在 Bug]]，并生成 [[系统级架构分析]] 报告。
*   **自动化代码审查：** 工作流脚本可以被配置为执行自动化的代码审查任务，识别代码风格问题、潜在的安全漏洞或性能瓶颈，并提供 [[代码审查]] 建议。

通过 [[外部工具集成]]，工作流脚本的能力可以进一步扩展，使其能够调用 [[第三方 CLI 工具]] 或其他外部服务，实现更灵活和强大的自动化能力。
