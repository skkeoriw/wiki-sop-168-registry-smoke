## 概念定义

Python 冒泡算法（Bubble Sort）是一种简单直观的排序算法。它的工作原理是重复地遍历待排序的列表，一次比较两个相邻的元素，如果它们的顺序错误（例如，在升序排序中，前一个元素大于后一个元素），就将它们交换过来。这个过程会一直重复，直到在一次完整的遍历中没有再发生任何元素交换，这意味着列表已经排序完成。冒泡算法的名字来源于较小（或较大）的元素会像气泡一样慢慢“浮”到列表的顶端（或底端）。尽管冒泡算法易于理解和实现，但其效率相对较低，尤其是在处理大量数据时，其时间复杂度为 O(n^2)，这使得它在实际应用中很少被用作主要的排序算法，更多地是作为教学示例或在数据量非常小的情况下使用。

## 技术细节

冒泡算法的核心在于嵌套循环。外层循环控制需要进行的遍历次数，最多需要进行 n-1 次遍历（其中 n 是列表的长度），因为在每次遍历中至少会将一个元素放到其最终位置。内层循环负责实际的比较和交换。在第 i 次外层循环遍历时，内层循环会从列表的第一个元素开始，一直比较到列表的倒数第 i 个元素，因为后面的 i 个元素已经是有序的了。

一个典型的 Python 实现如下：

```python
def bubble_sort(arr):
    n = len(arr)
    # 外层循环控制遍历次数
    for i in range(n):
        # 内层循环进行比较和交换
        # 优化：如果在一轮中没有发生交换，说明列表已经有序，可以提前结束
        swapped = False
        for j in range(0, n - i - 1):
            # 升序排序
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        # 如果本轮没有发生交换，则列表已排序
        if not swapped:
            break
    return arr
```

该算法的时间复杂度在最好情况下（列表已经有序）是 O(n)，因为只需要一次遍历来确认。在最坏情况下（列表完全逆序）和平均情况下，时间复杂度都是 O(n^2)。空间复杂度为 O(1)，因为它只需要常数额外的空间来存储临时变量。

## 应用场景

虽然冒泡算法的效率不高，但在某些特定场景下仍然有其价值。

1.  **教学与入门：** 由于其简单易懂的逻辑，冒泡算法是学习排序算法的绝佳起点，能够帮助初学者理解比较和交换的基本概念。
2.  **小型数据集：** 对于元素数量非常少（例如，少于 10 个）的数据集，冒泡算法的性能差异不明显，其实现简单可以节省开发时间。
3.  **特定优化场景：** 在某些情况下，如果数据基本有序，冒泡算法的优化版本（如上例中的 `swapped` 标志）可以快速完成排序。
4.  **作为其他算法的组成部分：** 在一些复杂的算法或数据结构中，可能会用到冒泡排序的思想来处理局部有序性。
5.  **演示与实时执行：** 在交互式编程环境（如 [[Open WebUI]]）中，冒泡算法可以被用来实时演示排序过程，例如在 [[Hermes Agent]] 的配置中，通过代码块运行（如 Python 冒泡算法实时执行）来展示其功能。
