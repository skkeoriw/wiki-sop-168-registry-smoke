## 模型类型

在 Claude Code 的动态工作流中，“模型类型”指的是用于执行工作流中不同阶段或任务的特定 AI 模型。Claude Code 支持多种模型，其中最主要的包括 Opus、Sonnet 和 Haiku。这些模型并非同质化存在，而是拥有不同的能力、速度和成本效益，从而允许用户根据具体需求进行灵活配置和选择。

Opus 通常被认为是旗舰级模型，拥有最强大的推理能力、最深入的理解力以及最广泛的知识储备，适用于需要高度准确性、复杂分析或最终报告合成等关键任务。然而，其计算成本也相对较高，处理速度可能不如其他模型。

Sonnet 模型则在性能和成本之间取得了较好的平衡，它能够处理相当复杂的任务，并且速度比 Opus 更快，成本也更低，因此常被用作分析和验证的中间阶段。

Haiku 模型则以其极高的速度和最低的成本著称，尽管其推理能力可能不如 Opus 或 Sonnet，但对于一些简单、重复性高或对实时性要求极高的任务，Haiku 是一个非常经济高效的选择。

通过“模型混合策略”，用户可以智能地组合使用这些不同类型的模型。例如，在工作流的初期规划和最终报告生成阶段使用 Opus 来确保质量和权威性，而在中间的数据处理、初步分析或验证阶段则切换到 Sonnet 或 Haiku，以优化成本和效率。这种策略能够有效弥补单一模型的局限性，实现性价比的最大化。

### 技术细节

动态工作流允许用户在配置任务时明确指定每个阶段或每个智能体（Agent）所使用的模型类型。这通常通过工作流的配置文件或直接在任务指令中进行声明。例如，一个工作流可能被设计为：

1.  **规划阶段（Opus）：** 负责理解用户意图，生成初步的工作流计划，并确定所需的智能体和外部工具。
2.  **数据提取与预处理阶段（Haiku/Sonnet）：** 快速拉取大量数据，并进行初步的清洗和格式化。
3.  **分析与验证阶段（Sonnet）：** 对提取的数据进行深入分析，可能涉及代码审查、关联分析或对抗验证。
4.  **最终合成与报告阶段（Opus）：** 将所有分析结果进行整合，生成最终的、高质量的报告或输出。

这种精细化的模型选择，使得工作流能够根据任务的复杂度和对成本、速度的要求进行动态调整。

### 应用场景

*   **成本优化：** 在处理海量数据或执行大量重复性任务时，优先使用 Haiku 或 Sonnet 模型，仅在关键节点使用 Opus，从而显著降低整体运行成本。
*   **性能提升：** 对于对响应速度要求极高的场景，例如实时代码审查或快速的初步分析，选择速度更快的 Sonnet 或 Haiku 模型可以显著缩短处理时间。
*   **复杂问题解决：** 在需要深度推理、复杂逻辑分析或生成权威性报告时，旗舰级模型 Opus 的强大能力是不可或缺的。
*   **[[多模型协作]]：** 动态工作流的核心优势之一就是能够无缝集成不同模型，让它们协同工作，发挥各自的优势，共同完成复杂任务。
*   **[[代码审查]]：** 在对代码进行初步审查时，可以使用 Sonnet 模型快速识别潜在问题，而在发现关键 Bug 或需要深入理解代码逻辑时，再切换到 Opus 进行更细致的分析。
