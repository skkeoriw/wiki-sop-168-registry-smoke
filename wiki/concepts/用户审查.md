## 概念定义

用户审查（User Audit）是 Claude Code 动态工作流（Dynamic Workflows）生命周期管理中的一个核心环节。在动态工作流启动后，系统会首先进入“规划”（Workflow Plan）阶段，生成一套详细的执行计划、流程图、工作流脚本以及预期的步骤。紧接着，便会进入“审计”（Audit）阶段，即用户审查阶段。在此阶段，用户被赋予了对 AI 自动生成的整个工作流进行全面审视和评估的权力。这包括但不限于检查工作流脚本的逻辑是否符合预期，每个步骤的执行内容是否准确，所选用的模型（如 [[Opus]]、[[Sonnet]]、[[Haiku]] 或其他 [[非 Claude 系列模型]]）是否恰当，以及预估的资源消耗（如智能体配额、计算时间）是否在可接受范围内。用户审查的目的是为了确保工作流的执行不会产生意外的后果，例如资源失控、数据丢失、不准确的分析结果，或者不符合预设的安全和成本要求。只有当用户对整个工作流计划满意并给予批准后，系统才会进入“执行”（Execution）阶段，开始大规模并发地运行智能体。这一机制极大地增强了用户对 AI 自动化过程的控制力，降低了潜在风险，并提升了工作流的可靠性和透明度。

## 技术细节

用户审查阶段是动态工作流“规划-审计-执行”模型中的关键中间步骤。在“规划”阶段，Claude Code 会根据用户输入的任务需求，自动生成一个详细的“工作流计划”（Workflow Plan）。这个计划包含了执行整个任务所需的全部步骤、子任务、调用的 [[外部工具]]（如 [[CodeGraph MCP]] 或 [[Codex CLI]]）、模型选择策略以及预估的 [[消耗]]。在进入“审计”阶段后，这个计划会以清晰易懂的格式呈现给用户。用户可以查看完整的 [[工作流脚本]]，理解每个 [[智能体]] 的行为逻辑，以及它们之间如何通过 [[多模型协作]] 或 [[对抗验证]] 来协同工作。特别重要的是，用户可以审查系统预估的 [[Agent 数量]] 和 [[预计 Agent 数量]]，以及可能涉及的 [[智能体配额]]。如果用户发现计划中的任何部分存在问题，例如模型选择不当、步骤逻辑错误、或者预估消耗过高，都可以选择修改或拒绝该计划，并要求 AI 重新生成。一旦用户批准了计划，系统才会进入“执行”阶段，并可能支持 [[大规模并发]] 执行，为每个 [[任务点]] 派生 [[独立智能体]]。

## 应用场景

用户审查机制在多种复杂软件工程任务中都至关重要。在进行大规模 [[代码审查]] 时，用户可以审查 AI 生成的审查计划，确保其覆盖了所有关键的检查点，并且不会对代码库造成不必要的负担。对于 [[GitHub Issue 分析]]，用户可以审查 AI 规划的分析路径，确认其是否能有效定位 [[核心 Bug]] 或 [[潜在 Bug]]，并评估其处理 [[崩溃数据]] 或 [[数据丢失]] 等问题的策略。在进行 [[存储库导航]] 或 [[代码导航]] 时，用户审查可以确保 AI 生成的导航路径是高效且符合预期的，避免 [[资源浪费]]。此外，在涉及 [[预算控制]] 的场景下，用户审查允许用户在工作流实际启动前，评估 [[成本与性能优化]] 的策略，并根据 [[性价比]] 需求调整 [[模型混合策略]]。通过用户审查，可以实现 [[自动化筛选]] 和 [[优先级排序]]，确保 AI 资源被用于最关键的任务，并防止 [[资源失控]]。
