## 概念定义

网页引用（Webpage Citation）是 [[Open WebUI]] 提供的一项重要功能，它允许用户在与 [[Hermes Agent]] 或其他兼容的 AI Agent 进行交互时，将特定的网页内容作为上下文信息提供给 Agent。这项功能极大地扩展了 Agent 的信息获取和处理能力，使其能够超越预设的知识库或用户输入的文本，直接从互联网上检索和理解实时信息。通过网页引用，用户可以引导 Agent 关注特定的网页链接，并从中提取关键信息、进行分析、总结内容，甚至基于网页内容生成回复或执行任务。这对于需要处理最新资讯、研究特定主题、或者需要 Agent 理解外部文档的场景尤为关键。它克服了传统聊天工具仅能处理用户直接输入文本的局限性，使得 AI Agent 能够更深入、更全面地理解用户意图和外部世界。

## 技术细节

在 [[Open WebUI]] 中实现网页引用功能，通常涉及以下几个技术层面：

1.  **URL 输入与解析：** 用户可以通过界面输入一个或多个网页的 URL。Open WebUI 会接收这些 URL，并可能进行初步的验证，确保其格式正确。
2.  **网页内容抓取：** Agent 的后端服务（例如通过 [[Hermes Agent]] 的配置）会负责抓取指定 URL 的网页内容。这可能涉及到使用 HTTP 请求库来获取 HTML 源代码。
3.  **内容提取与清洗：** 原始的 HTML 源代码包含了大量的标记语言和脚本，对于 AI Agent 来说并不直接可用。因此，需要一个内容提取器来解析 HTML，去除不相关的标签（如广告、导航栏、页脚等），并提取出主要的文本内容。一些高级的实现还会尝试提取图片、表格等结构化信息。
4.  **Markdown 支持与限制：** 尽管 Open WebUI 支持流式输出和 [[Markdown 解析]]，但对于网页内容的解析，可能存在一定的限制。复杂的网页结构、动态加载的内容（JavaScript 渲染的页面）或反爬虫机制都可能影响内容的准确抓取和提取。
5.  **上下文整合：** 提取出的网页内容会被整合到 Agent 的对话上下文中。这使得 Agent 在后续的生成或决策过程中，能够参考这些外部信息。
6.  **跨端访问集成：** 如 [[Hermes Agent]] 的配置指南所述，通过配置本地 IP 和端口，用户可以在手机浏览器上与电脑端的 Agent 交互，并支持文件上传、截图引用及网页引用，这表明网页引用功能是跨端访问能力的一部分，能够无缝集成到多设备交互流程中。

## 应用场景

网页引用功能在多种场景下都能发挥重要作用：

*   **信息研究与总结：** 用户可以提供一篇新闻报道、一篇学术论文或一个产品评测的链接，要求 Agent 总结其核心观点、提取关键数据或进行对比分析。
*   **实时信息查询：** 当用户需要了解最新的股票行情、天气预报或某个事件的进展时，可以直接提供相关网页链接，让 Agent 获取并播报最新信息。
*   **辅助决策：** 在购物、旅行规划等场景中，用户可以提供多个产品页面或酒店预订页面的链接，让 Agent 比较价格、功能、用户评价等，从而帮助用户做出更明智的决策。
*   **内容创作辅助：** 对于需要撰写文章、博客或报告的用户，可以提供参考网页，让 Agent 提取相关素材、提供写作灵感或检查事实准确性。
*   **技术文档查阅：** 在开发过程中，开发者可以提供 API 文档、技术论坛帖子的链接，让 Agent 帮助查找特定函数的使用方法、解决遇到的技术问题。
*   **[[Hermes Agent]] 的视觉与网页处理：** 在 [[Hermes Agent]] 的主副模型架构中，网页处理被归类为辅助任务，通常由高性价比的副模型（如 [[Gemini 1.5 Flash]]）处理，这表明网页引用是 Agent 处理复杂信息和执行特定任务的重要组成部分，旨在实现性能与成本的最佳平衡。

网页引用功能极大地提升了 AI Agent 的实用性和智能化水平，使其能够更有效地与外部世界互动，为用户提供更丰富、更精准的服务。
