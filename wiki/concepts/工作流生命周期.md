## 工作流生命周期

工作流生命周期是指一个动态工作流从概念生成到最终完成的整个过程。它定义了工作流的演进路径，确保了任务的有序、可控和高效执行。在 Claude Code 的动态工作流中，这一生命周期被清晰地划分为“规划-审计-执行”三个核心阶段，为用户提供了对复杂任务的深度参与和控制能力。

### 概念定义

工作流生命周期是动态工作流的核心管理机制，它确保了从任务的初步构思到最终结果产出的每一个环节都得到妥善处理。这个生命周期不仅仅是任务的顺序执行，更包含了智能的规划、严格的审计和高效的执行。在“规划”阶段，系统会预先生成详细的执行计划和流程图，明确任务的分解、所需资源以及预期的输出，但此时并不会立即启动实际的计算过程。紧接着是“审计”阶段，这是用户介入的关键环节。用户可以详细审查工作流的脚本、每一个执行步骤、将要使用的模型（如 [[Opus]]、[[Sonnet]]、[[Haiku]] 等）以及预估的成本消耗。这一阶段的设计旨在赋予用户最终的控制权，允许其在执行前对工作流进行调整、优化或批准。一旦通过审计，工作流便进入“执行”阶段，此时系统能够支持大规模的并发处理，例如，同时启动数十个 [[子智能体]] 来并行处理大量的任务点，极大地提升了处理效率。整个生命周期强调了透明度、用户控制和可扩展性，使得动态工作流能够胜任从大规模 [[GitHub Issue 分析]] 到复杂的 [[代码审查]] 等多种高级应用场景。

### 技术细节

动态工作流的生命周期管理是其核心价值所在。在最新版本的 Claude Code 中，激活动态工作流的指令统一为 `ultra code`。一旦激活，工作流便遵循“规划-审计-执行”的路径：

1.  **规划 (Workflow Plan)：** 系统首先生成一个详细的执行计划和流程图，明确任务的分解、步骤、依赖关系以及预期的资源消耗。这一阶段的产出是工作流的蓝图，但不涉及实际的计算或资源占用。
2.  **审计 (Audit)：** 用户在此阶段拥有完全的审查权。可以详细检查工作流脚本、每一个执行步骤、将要使用的模型类型（包括 [[旗舰模型]] 和其他模型系列）、以及预估的成本消耗（如 [[智能体配额]]）。用户可以根据审计结果决定是否继续执行、进行修改或取消工作流。
3.  **执行 (Execution)：** 经过用户批准后，工作流进入执行阶段。此阶段支持大规模并发处理，能够同时启动多个 [[子智能体]] 来并行处理任务，例如，在处理 50 个 [[GitHub Issue]] 时，系统可以同时启动数十个智能体实例，极大地提高了处理效率。

### 应用场景

工作流生命周期管理使得动态工作流在处理复杂和大规模任务时表现出色。

*   **大规模 GitHub Issue 分析与验证：** 对于拥有大量待处理 Issue 的开源项目，工作流生命周期能够实现自动化筛选，识别出 [[T10 级别 Bug]] 等关键问题。通过“规划-审计-执行”的流程，可以将任务拆解为分析、验证和报告合成等阶段，确保处理的准确性和效率。用户可以在审计阶段审查分析计划和使用的模型，控制 [[预算控制]]，避免 [[资源失控]]。
*   **代码审查与重构：** 在代码审查过程中，工作流可以被设计为先进行初步的代码静态分析（规划），然后由用户审计分析结果和建议（审计），最后由多个 [[子智能体]] 并行执行代码重构或生成修复建议（执行）。这种分阶段的处理方式能够提高代码审查的质量和效率，并降低 [[模型局限性]] 带来的风险。
