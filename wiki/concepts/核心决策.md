# 核心决策

在 [[Hermes Agent]] 的主副模型架构中，**核心决策**是指由主模型承担的、处理复杂逻辑和关键判断的任务。这与由副模型处理的辅助任务（如批准、压缩、重刷记忆、MCP 调用、Session 搜索、视觉及网页处理等）形成鲜明对比。主模型负责理解和执行 Agent 的核心指令，进行深入分析，并做出最终的判断或规划，是 Agent 实现高级智能和生产力的关键所在。

## 技术细节

主模型在 [[Hermes Agent]] 中扮演着“大脑”的角色，它需要具备强大的理解能力、推理能力和决策能力。当用户输入一个复杂的指令或需要 Agent 进行深度分析时，这个任务会被分配给主模型。主模型会调用其内部的知识库和推理引擎，结合当前对话的上下文信息，来生成最优的解决方案或执行计划。例如，在进行长线任务规划、复杂问题分析、或者需要 Agent 做出权衡和选择时，主模型都会发挥其核心作用。

## 应用场景

核心决策的应用场景非常广泛，主要体现在以下几个方面：

*   **复杂任务规划与执行：** 当用户提出一个需要多步骤、多模块协同完成的任务时，主模型需要进行整体规划，并决定各个子任务的执行顺序和方式。
*   **深度分析与洞察：** 对于需要 Agent 进行深入分析、提取关键信息、或者提供专业见解的场景，主模型会负责进行深度推理和信息整合。
*   **策略制定与权衡：** 在面临多种选择或需要权衡利弊时，主模型需要根据预设的策略或用户偏好，做出最优的决策。
*   **异常处理与容错：** 当遇到意外情况或任务执行过程中出现问题时，主模型需要分析原因并制定相应的应对策略。

通过将复杂逻辑和核心决策交给主模型，并利用高性价比的副模型处理辅助任务，[[Hermes Agent]] 实现了 [[Token 优化配置]] 和成本效益的最佳平衡，同时保证了 Agent 的高级功能和智能性。
