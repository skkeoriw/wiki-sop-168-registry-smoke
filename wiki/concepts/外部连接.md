# 外部连接

外部连接（External Connections）是 [[Open WebUI]] 的一项重要配置功能，它允许用户将 [[Open WebUI]] 与外部的 API 服务进行集成。通过配置外部连接，用户可以指定一个外部服务的 URL、API 密钥以及兼容性设置，从而使 [[Open WebUI]] 能够调用这些外部服务来执行任务。这对于利用不同的语言模型、AI 服务或自定义后端服务至关重要，尤其是在成本优化和功能扩展方面。

## 技术细节

在 [[Open WebUI]] 的管理员设置中，添加外部连接时需要输入以下关键参数：

*   **URL**: 这是外部 API 服务的访问地址。例如，配置 [[Hermes Agent]] 时，通常会指向其本地运行的 API 端点，如 `http://localhost:8642/v1`。这个 URL 指明了 [[Open WebUI]] 将请求发送到何处。
*   **API 密钥**: 用于验证用户身份和授权访问外部服务。这个密钥通常由用户在服务提供商处生成或在本地配置文件中自定义。它确保只有授权用户才能访问和使用外部服务。
*   **兼容性**: 指明了外部 API 服务所遵循的接口标准。[[Open WebUI]] 支持兼容 [[OpenAPI]] 接口的服务，这意味着它可以与遵循 OpenAPI 规范的各种 AI 模型和 API 进行无缝集成。

通过这种方式，[[Open WebUI]] 能够作为一个统一的交互界面，调用不同的后端模型或服务，实现 [[主副模型架构]] 的灵活性。例如，可以将复杂的任务交给主模型处理，而将一些相对简单的任务（如审批、压缩、记忆刷新、MCP 调用与技能处理、视觉与网页任务等）分配给成本更低的 [[副模型]]（Auxiliary Tasks）来完成，从而实现 [[Token 优化配置]] 和 [[成本优化]]。

## 应用场景

外部连接的配置极大地扩展了 [[Open WebUI]] 的应用场景：

1.  **模型切换与成本控制**: 用户可以轻松地在不同的语言模型之间切换，例如使用 [[Gemini 1.5 Flash]] 作为副模型来处理低成本的辅助任务，而将更复杂的任务交给更强大的主模型。这有助于在保证性能的同时，显著降低 [[Token 消耗]]。
2.  **集成第三方服务**: 除了语言模型，还可以集成其他提供 API 服务的第三方工具或平台，实现更丰富的功能，如数据分析、内容生成、自动化流程等。
3.  **本地与云端混合部署**: 用户可以将本地运行的模型（如通过 [[Ollama]] 部署的模型）通过外部连接集成到 [[Open WebUI]] 中，同时也可以连接到云端的 AI 服务。这种混合部署模式提供了灵活性和可扩展性。
4.  **定制化 Agent**: 通过配置外部连接，可以构建高度定制化的 [[Hermes Agent]] 或其他 AI Agent，使其能够调用特定的技能和工具，满足特定的工作流程需求。

总而言之，外部连接是实现 [[Open WebUI]] 高级功能和灵活性的关键配置项，它使得用户能够更有效地管理 AI 资源，优化成本，并扩展 AI 应用的能力边界。
