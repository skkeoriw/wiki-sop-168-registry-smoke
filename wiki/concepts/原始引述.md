## 技术细节

原始引述揭示了 [[Hermes Agent]] 在技术实现上的几个关键点：

*   **主副模型架构：** Agent 支持配置主模型和副模型，主模型负责处理复杂逻辑和核心决策，而副模型（辅助任务）则处理批准、压缩、重刷记忆、MCP 调用、Session 搜索、视觉及网页处理等辅助性任务。这种分工旨在优化 [[Token 消耗]]。
*   **模型选择建议：** 针对副模型，建议选择高性价比模型（如 [[Gemini 1.5 Flash]]），以实现性能与成本的最佳平衡。
*   **交互界面限制：** 在传统的聊天工具中使用 [[Hermes Agent]] 存在局限性，主要是因为许多聊天工具不支持 Markdown 解析，这影响了用户体验。因此，推荐使用 [[Open WebUI]] 等支持 Markdown 解析的界面。
*   **稳定性优势：** 相较于其他框架（如 Open Cloud），[[Hermes Agent]] 在稳定性方面表现更优，不易出现崩溃或引入 Bug。

## 应用场景

原始引述暗示了 [[Hermes Agent]] 的多种应用场景：

*   **生产力工具：** 通过主副模型架构和成本优化，使其适用于需要处理大量任务的长线任务和生产环境部署。
*   **本地化部署：** 尽管未直接提及，但其稳定性优势和对 [[Open WebUI]] 的兼容性，使其能够与本地模型（如通过 [[Ollama]] 部署的模型）结合，实现本地化 Agent 服务。
*   **交互体验升级：** 推荐使用 [[Open WebUI]] 等工具，旨在提升用户与 Agent 交互的体验，克服传统聊天工具的限制。
*   **成本控制：** 通过将简单任务分配给低成本副模型，有效控制 [[Token 消耗]]，降低运营成本。
