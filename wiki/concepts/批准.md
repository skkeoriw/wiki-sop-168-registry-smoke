# 批准

在 [[Hermes Agent]] 的主副模型架构中，“批准”特指一种**辅助任务**，它被设计用来处理相对简单、不涉及复杂逻辑或核心决策的工作。这种任务分工策略的核心目标是实现 [[Token 优化配置]]，通过将这类辅助性任务分配给成本效益更高的模型（例如 [[Gemini 1.5 Flash]]），从而显著降低整体的 Token 消耗和运营成本，达到 [[性能与成本平衡]] 的最佳状态。

## 技术细节

“批准”作为辅助任务的一部分，通常与以下任务类型相关联：

*   **压缩：** 对数据或信息进行压缩处理。
*   **重刷记忆：** 更新或重置 Agent 的记忆模块。
*   **MCP 调用：** 执行特定的消息通信协议调用。
*   **Session 搜索：** 在当前会话中进行信息检索。
*   **视觉及网页处理：** 对图像内容进行分析或处理网页信息。

这些任务的共同特点是它们通常不需要 Agent 进行深度推理或复杂的逻辑判断，而是执行预设的、相对标准化的操作。通过将这些任务分配给副模型，主模型可以更专注于处理需要高度智能和复杂分析的核心任务，例如 [[复杂逻辑]] 和 [[核心决策]]。这种架构设计是 [[Hermes Agent]] 在面对 [[Token 消耗]] 过高问题时提出的解决方案之一，旨在通过精细化的 [[任务分工]] 来提升效率和降低成本。

## 应用场景

“批准”任务的应用场景非常广泛，尤其是在需要高效处理大量信息或执行重复性操作的场景下。例如：

*   **内容审核与过滤：** 副模型可以快速“批准”或标记不符合特定规则的内容，减少人工干预。
*   **数据预处理：** 在将数据输入主模型进行深度分析前，副模型可以负责数据的初步清洗、格式转换或压缩。
*   **用户交互辅助：** 在 [[Open WebUI]] 等界面中，副模型可以快速响应用户的简单指令，如搜索历史记录、压缩对话内容等。
*   **自动化流程：** 在自动化工作流中，“批准”可以作为某个步骤的确认环节，由副模型快速完成。

通过将“批准”等辅助任务交给更经济的模型，[[Hermes Agent]] 能够为用户提供更具成本效益的智能服务，尤其是在 [[生产环境部署]] 或需要处理大量请求的场景下，这种优化显得尤为重要。这与 [[Opencloud]] 等其他框架在成本控制方面的考量有所不同，Hermes Agent 提供了更灵活的 [[主副模型架构]] 来应对这一挑战。
