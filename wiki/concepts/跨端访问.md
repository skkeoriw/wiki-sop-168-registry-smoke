## 概念定义

跨端访问（Cross-Device Access）是一种技术能力，它允许用户在不同的计算设备上，通过网络连接，无缝地访问和控制同一个应用程序或服务。在 [[Hermes Agent]] 的语境下，跨端访问主要体现在用户可以通过手机、平板电脑等移动设备上的浏览器，直接与部署在本地电脑上的 Agent 进行交互。这种能力极大地提升了用户体验和灵活性，使得 Agent 不再局限于本地电脑屏幕，而是可以随时随地进行操作和管理。它打破了设备间的壁垒，将本地计算资源的能力延伸到更广泛的访问场景，用户无需将 Agent 部署到云端，即可享受到便捷的远程控制，这对于家庭使用或需要灵活部署的场景尤为重要。通过配置本地 IP 地址和端口，用户可以轻松地在手机浏览器中输入相应的地址，从而连接到本地运行的 Agent 服务，实现文件上传、截图引用、网页引用等多种交互功能，极大地扩展了 Agent 的使用边界和便利性。

## 技术细节

实现跨端访问的核心在于建立一个可供外部访问的网络接口。在 [[Hermes Agent]] 的配置中，通常需要通过 `config.yaml` 文件来启用 API 服务，例如设置 `enable_api: true`。同时，为了安全起见，通常还需要配置 API 访问密码。一旦 API 服务被启用并配置妥当，Agent 就会监听指定的本地 IP 地址和端口（例如 8080）。用户随后可以在其他设备（如手机）的浏览器中，输入该本地 IP 地址和端口，即可访问 Agent 的 Web UI 界面。例如，通过 [[Open WebUI]]，用户可以将接口设置为 `http://localhost:8642/v1`（这里的端口可能因具体配置而异），从而实现与本地 Agent 的通信。这种机制允许 Agent 的后端服务暴露给局域网内的其他设备，从而实现远程交互，而无需将 Agent 部署到云端服务器，也不占用本地硬件资源。

## 应用场景

跨端访问最直接的应用场景是提升用户与本地部署的 [[Hermes Agent]] 的交互体验。许多用户习惯在微信等聊天工具中使用 Agent，但这些工具可能存在 Markdown 解析支持差、多轮对话管理混乱等问题。通过 [[Open WebUI]] 提供的跨端访问能力，用户可以在手机浏览器上直接与电脑端的 Agent 交互，如同使用 ChatGPT 一样，拥有侧边栏历史记录、流式输出、语音发音等功能。这使得 Agent 的使用更加便捷和高效。例如，用户可以在电脑上运行 Agent，然后在手机上通过浏览器上传文件、进行截图引用或网页引用，实现更丰富的交互。此外，对于需要远程控制或在不同房间、不同设备间切换操作的用户来说，跨端访问也提供了极大的便利性，例如在家庭环境中，可以在客厅的电脑上运行 Agent，然后在卧室的手机上进行操作。
