## 模型系列

模型系列（Model Series）是指在人工智能系统中，一组被设计、训练或配置用于执行特定类型任务或具备不同能力水平的模型集合。这些模型通常具有层级结构或功能区分，可以根据任务的复杂性、资源限制或性能需求进行选择和组合。在 Claude Code 的动态工作流（Dynamic Workflows）中，模型系列的概念尤为重要，因为它允许系统根据任务需求动态地调用不同的模型来执行子任务，从而实现高效且成本优化的解决方案。

### 技术细节

Claude Code 的动态工作流支持调用不同的模型系列，例如 [[Opus]]（旗舰级模型，通常用于复杂推理和高级分析）、[[Sonnet]]（平衡性能与成本的模型）以及 [[Haiku]]（轻量级模型，适用于快速响应和低成本任务）。工作流的“规划”阶段会根据任务的性质（如 Bug 分析的深度、代码审查的范围）来选择最适合的模型系列。例如，对于需要深入理解代码逻辑和潜在 Bug 的复杂任务，可能会优先选择 Opus；而对于简单的代码元数据抓取或初步筛选，则可能使用 Haiku 以节省成本。这种模型系列的灵活配置是实现“性价比最大化”的关键。在“审计”阶段，用户可以看到工作流计划中预期的模型系列及其预计消耗，从而进行有效的预算控制。

### 应用场景

在 [[Claude Code Dynamic Workflows]] 的高级应用场景中，模型系列的应用体现在多个方面。例如，在处理大规模的 [[GitHub Issue 分析]] 时，系统可以先使用一个轻量级的模型系列（如 Haiku）对所有 Issue 进行初步的自动化筛选和优先级排序，识别出潜在的“核心 Bug”或高优先级问题。随后，对于这些被标记为高优先级的 Issue，再调用更强大的模型系列（如 Opus）进行深入的分析，包括代码逻辑的敏感度分析、潜在的数据丢失或崩溃数据等风险评估。这种分阶段处理的策略，能够显著提高效率并优化资源利用。此外，在 [[代码审查]] 场景中，不同的模型系列可以承担不同的角色：一个模型系列负责检查代码风格和规范，另一个模型系列则专注于发现潜在的逻辑错误或安全漏洞。通过 [[多模型协作]]，可以实现 [[三方协同]]，结合不同模型的 [[互补优势]]，生成更全面、准确的审查报告。
