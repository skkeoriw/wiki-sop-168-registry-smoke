## 概念定义

Plan 预览（Plan Preview）是 Claude Code 动态工作流（Dynamic Workflows）中的一项核心功能，它提供了一个在任务执行前的关键审查和批准环节。在用户输入一个任务指令后，动态工作流系统会首先生成一个详细的执行计划，而 Plan 预览功能则允许用户在工作流正式启动之前，对这个计划进行细致的审视。这包括但不限于：预估将要调用的 [[智能体指挥官]] 和 [[子智能体]] 的数量，以及每个阶段或每个 Agent 将会使用的 [[模型类型]]（例如 [[Opus]]、[[Sonnet]] 或 [[Haiku]]）。通过强制开启 Plan 预览，用户可以确保工作流的执行符合预期，避免不必要的 [[消耗]] 或 [[资源失控]]。这一机制极大地增强了用户对复杂自动化流程的控制力，使得用户能够主动管理 [[智能体配额]] 和 [[预算控制]]，从而实现 [[性价比]] 的最大化。它本质上是一种“批准前不执行”的策略，确保了透明度和可控性，是实现高效、安全和成本效益高的动态工作流的关键组成部分。

## 技术细节

Plan 预览功能通过在动态工作流的生命周期（[[工作流生命周期]]）中引入一个“计划生成”和“计划审查”的阶段来实现。当用户提交一个任务时，Claude Code 的核心引擎会解析任务需求，并根据预设的 [[工作流关键词]] 和内部逻辑，规划出一系列 Agent 的调用顺序、数据流向以及所需的 [[外部工具]] 集成。这个规划过程会输出一个结构化的执行计划，其中详细列出了每个步骤将要激活的 Agent 数量（[[预计 Agent 数量]]）以及它们将使用的模型（[[模型类型]]）。

用户可以通过特定的指令或配置来强制启用 Plan 预览。例如，在输入任务时明确要求“在批准前不要开始执行”。一旦计划生成，系统会以清晰易懂的格式向用户展示这个计划，允许用户进行 [[审计]]。用户可以审查计划的合理性，判断是否需要调整 Agent 的数量、模型选择，或者是否需要引入 [[第三方 CLI 工具]] 来弥补当前模型的 [[模型局限性]]。如果用户对计划满意，则可以批准执行；如果不满意，则可以要求系统重新规划或进行修改。

此外，动态工作流生成的 `.js` [[工作流脚本]] 可以在 Plan 预览阶段被审查，这有助于开发者理解 Agent 是如何驱动 [[外部工具集成]] 的，例如集成 [[Codex CLI]] 进行 [[代码审查]]。这种透明度对于理解和优化工作流至关重要。

## 应用场景

Plan 预览功能在多种场景下都显得尤为重要：

1.  **成本与性能优化：** 在执行成本较高的任务时，用户可以通过 Plan 预览来评估 [[性价比]]。例如，在分析大量代码或数据时，用户可以审查是否所有阶段都使用了 [[旗舰模型]]，并可能指示系统在中间阶段使用更经济的模型（如 [[Haiku]] 或 [[Sonnet]]），仅在关键的规划或 [[最终合成]] 阶段使用 [[Opus]]。这有助于实现 [[成本与性能优化]]。
2.  **复杂任务的控制：** 对于涉及多步骤、多 Agent 协作的复杂任务，如 [[GitHub Issue 分析]] 或 [[系统级架构分析]]，Plan 预览允许用户在执行前对整个任务的分解和执行路径进行把控，确保 [[分析深度]] 和 [[准确性]] 符合预期。
3.  **防止资源浪费和意外执行：** 在不确定任务执行结果或担心 [[资源浪费]] 时，Plan 预览提供了一个安全网。用户可以确认计划是否会触发不必要的 [[并发执行]] 或 [[并行分析]]，从而避免 [[资源失控]]。
4.  **理解 Agent 的工作方式：** 对于希望深入了解 Claude Code 如何工作的用户，审查 Plan 预览中的 Agent 分配和模型选择，可以帮助他们理解 [[Agent 驱动]] 的机制以及 [[多模型协作]] 的具体实现。
5.  **集成外部工具的验证：** 当工作流需要集成 [[Codex CLI]] 或 [[CodeGraph MCP]] 等 [[第三方 CLI 工具]] 时，Plan 预览可以展示这些工具在计划中的位置和作用，确保其被正确调用以实现 [[Codex CLI 交叉验证]] 或 [[CodeGraph MCP 集成]]。

通过强制开启 Plan 预览，用户能够更主动地参与到自动化流程的设计和执行中，将 [[智能体指挥官]] 的能力与用户的领域知识和业务需求更紧密地结合起来。

## 实践洞察与行动建议

1.  **采用“模型混合策略”：** 建议在分析和验证的中间阶段使用 Haiku 或 Sonnet 模型，仅在最初规划和最终报告合成时使用旗舰级的 Opus 模型，以最大化性价比。
2.  **强制开启 Plan 预览：** 在输入任务时，明确要求“在批准前不要开始执行”，并要求展示“每个阶段预计使用的 Agent 数量”和“模型类型”。
3.  **脚本复用：** 动态工作流生成的 `.js` 脚本可以保存并复用于其他项目。在 VS Code 中审查这些脚本，可以帮助开发者理解 Agent 是如何驱动外部工具（如 Codex）的。
4.  **利用 MCP 扩展边界：** 结合 CodeGraph 等 MCP 工具，可以将 Claude Code 的能力从简单的对话扩展到复杂的系统级架构分析。
